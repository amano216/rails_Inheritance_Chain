🗄️ ActiveRecord - データベース関連の全機能
ActiveRecord
(データベース担当)		 ActiveRecord::Base
(モデルの親)		 find, where, create, update, destroy, save, valid? データベースのデータを操作するための基本機能。すべてのモデルクラスはこれを継承します。
ActiveRecord::Associations
(関連系)		 belongs_to, has_one, has_many, has_many :through 1対1、1対多の関係を定義。例：ユーザーが複数の投稿を持つ。
has_and_belongs_to_many 多対多の関係を定義。例：投稿とタグの関係。
ActiveRecord::Migration
(設計図の管理者)		 create_table, drop_table, rename_table テーブルの作成・削除・名前変更。データベースの構造を変更します。
add_column, remove_column, change_column, add_index カラム（列）の追加・削除・変更、インデックスの追加。
ActiveRecord::Scope
(検索のショートカット)		 scope, default_scope よく使う検索条件に名前を付けて再利用。例：scope :published, -> { where(published: true) }
ActiveRecord::Validations
(検証系)		 validates, validates_presence_of, validates_uniqueness_of データの妥当性チェック。空でないか、重複していないかなどを検証。
validates_length_of, validates_format_of, validates_inclusion_of 文字数、形式、選択肢の範囲などの詳細な検証。
ActiveRecord::Callbacks
(コールバック)		 before_validation, after_validation, before_save, after_save データ保存の各段階で自動実行される処理を定義。
before_create, after_create, before_destroy, after_destroy 作成時・削除時に特定の処理を自動実行。
ActiveRecord::QueryMethods
(クエリメソッド群)		 select, joins, includes, eager_load, preload N+1問題を防ぐための関連データの効率的な読み込み。
group, having, order, limit, offset, distinct グループ化、並び替え、件数制限など、SQLの高度な操作。
pluck, pick, ids, count, sum, average, minimum, maximum 特定カラムの値取得や集計計算を効率的に実行。
ActiveRecord::Transactions
(トランザクション)		 transaction, after_commit, after_rollback 複数の処理をまとめて実行し、エラー時は全て取り消す仕組み。
ActiveRecord::Enum
(列挙型)		 enum status: { draft: 0, published: 1 } ステータスなど決まった値を、分かりやすい名前で扱う。
ActiveRecord::Store
(ストア)		 store, store_accessor JSON形式でデータを1つのカラムに保存し、複数の属性として扱う。
ActiveRecord::Locking
(ロック機能)		 lock, with_lock, optimistic_locking 同時更新を防ぐためのデータロック機能。
ActiveRecord::Serialization
(シリアライズ)		 serialize, serialized_attributes 配列やハッシュなどのRubyオブジェクトをデータベースに保存。
ActiveRecord::Relation
(リレーション)		 to_sql, explain, exists?, any?, none?, one?, many? SQLの確認、クエリの実行計画、レコードの存在チェック。
ActiveRecord::Schema
(スキーマ定義)		 create_join_table, add_foreign_key, remove_foreign_key 結合テーブルの作成、外部キー制約の管理。
🎮 ActionController - リクエスト処理の全機能
ActionController
(司令塔担当)		 ActionController::Base
(コントローラーの親)		 render, redirect_to, params, session, flash, cookies ユーザーのリクエストに応じて処理を振り分ける司令塔の基本機能。
ActionController::Filters
(フィルター系)		 before_action, after_action, around_action アクションの前後に共通処理を実行。例：ログインチェック。
skip_before_action, only, except 特定のアクションだけフィルターを除外。ログイン画面でログインチェックをスキップなど。
ActionController::StrongParameters
(データ保護系)		 require, permit, fetch Mass Assignment攻撃を防ぐため、許可されたパラメータのみを受け取る。
ActionController::RequestForgeryProtection
(セキュリティ)		 protect_from_forgery, form_authenticity_token CSRF攻撃を防ぐためのセキュリティ機能。通常は自動で設定される。
ActionDispatch::Request
(リクエスト情報)		 get?, post?, remote_ip, user_agent, xhr? ユーザーからのリクエストの詳細情報。requestという変数でアクセス。
ActionController::Streaming
(ストリーミング)		 send_file, send_data, stream ファイルやデータをブラウザに送信。大きなファイルの効率的な配信。
ActionController::Caching
(キャッシュ)		 caches_page, caches_action, cache 処理結果を一時保存して高速化。
expire_page, expire_action, expire_fragment 保存されたキャッシュを削除して最新情報を表示。
ActionController::Rescue
(エラー処理)		 rescue_from, rescue_with_handler 特定の例外が発生した時の処理を定義。エラー画面のカスタマイズ。
ActionController::HttpAuthentication
(認証)		 authenticate_with_http_basic, authenticate_with_http_token HTTPレベルでのユーザー認証を実装。
ActionController::Cookies
(クッキー)		 cookies.permanent, cookies.signed, cookies.encrypted クッキーの保存期間やセキュリティレベルを制御。
ActionController::Head
(ヘッダー)		 head, headers HTTPヘッダーのみを返す。APIで使用。
ActionController::Renderers
(レンダラー)		 respond_to, format.json, format.xml リクエストの形式に応じて異なる形式でレスポンスを返す。
🎨 ActionView - ビュー生成の全機能
ActionView
(見た目担当)		 ActionView::Base
(ビューの親)		 (ヘルパーメソッド群) HTMLを生成するビューファイル（.html.erb）の基盤機能。
ActionView::Helpers
(便利道具モジュール)		 link_to, image_tag, form_with, button_to HTMLタグを簡単に作るための便利メソッド群。
number_to_currency, number_to_percentage, number_with_delimiter 数値を通貨表示や％表示など、見やすい形式に変換。
ActionView::Layouts
(レイアウト機能)		 layout, content_for, yield 共通レイアウトと各ページ固有のコンテンツを組み合わせる仕組み。
ActionView::Partials
(部分テンプレート)		 render partial:, collection:, locals: HTMLの一部を別ファイルに分けて再利用。
ActionView::FormBuilder
(フォームビルダー)		 text_field, password_field, text_area, check_box, radio_button 様々な種類の入力フォームを生成。
select, collection_select, date_select, time_select ドロップダウンや日時選択フォームを生成。
ActionView::AssetTagHelper
(アセットタグ)		 javascript_include_tag, stylesheet_link_tag, asset_path JavaScriptやCSSファイルを読み込むタグを生成。
ActionView::SanitizeHelper
(サニタイズ)		 sanitize, strip_tags, strip_links XSS攻撃を防ぐため、危険なHTMLタグやJavaScriptを除去。
ActionView::TextHelper
(テキスト加工)		 truncate, excerpt, highlight, simple_format, pluralize テキストの表示を整形・加工するヘルパーメソッド群。
ActionView::DateHelper
(日付ヘルパー)		 distance_of_time_in_words, time_ago_in_words 「3日前」「約1時間前」のような人間的な時間表現。
📧 ActionMailer - メール送信の全機能
ActionMailer
(手紙担当)		 ActionMailer::Base
(メーラーの親)		 mail, deliver_now, deliver_later メールを作成・送信する基本機能。コントローラーと似た構造。
mail
(メール作成)		 to:, from:, subject:, cc:, bcc: メールの宛先、件名、送信元などを設定。
attachments
(添付ファイル)		 attachments['filename.jpg'] = File.read(...) メールにファイルを添付する機能。
ActionMailer::Preview
(プレビュー)		 preview 開発環境でメールの見た目を確認する機能。
ActionMailer::DeliveryMethods
(配信方法)		 smtp, sendmail, test, file メールの送信方法を設定（SMTP、テスト用など）。
⚙️ ActiveJob - 非同期処理の全機能
ActiveJob
(重い仕事担当)		 ActiveJob::Base
(ジョブの親)		 perform_later, perform_now, set 時間のかかる処理をバックグラウンドで実行。
queue_as
(キュー管理)		 queue_as :urgent, queue_as :default ジョブの優先度に応じて待機列を指定。
ActiveJob::Callbacks
(コールバック)		 before_enqueue, after_enqueue, before_perform, after_perform ジョブの各段階で自動実行される処理。
ActiveJob::Exceptions
(例外処理)		 retry_on, discard_on, retry_job ジョブ失敗時の再試行ロジックやエラーハンドリング。
ActiveJob::QueueAdapters
(アダプター)		 Sidekiq, Resque, DelayedJob, Async 様々なバックグラウンドジョブ処理システムとの連携。
🔧 ActiveSupport - 便利機能の全集合
ActiveSupport
(便利道具担当)		 ActiveSupport::Concern
(機能追加モジュール)		 included, class_methods, prepended 複数のクラスで共通して使いたい機能をモジュール化。
Objectへの拡張機能 blank?, present?, try, try!, in? Rubyの基本機能を拡張。nilや空文字列の判定など。
Date, Timeへの拡張機能 ago, since, beginning_of_day, end_of_month, yesterday, tomorrow 3.days.ago（3日前）のような直感的な日時操作。
ActiveSupport::Inflector
(語形変化)		 pluralize, singularize, camelize, underscore, humanize 文字列の形式を自動変換（user → Users、user_comment → UserComment）。
ActiveSupport::Cache
(キャッシュストア)		 fetch, read, write, delete, clear, increment メモリやRedisなど、様々なキャッシュストアを統一的に扱う。
ActiveSupport::Notifications
(イベント通知)		 instrument, subscribe, unsubscribe アプリケーション内のイベントを監視・計測。
ActiveSupport::Duration
(期間)		 seconds, minutes, hours, days, weeks, months, years 時間の計算を直感的に行える（例：3.days + 2.hours）。
ActiveSupport::TimeZone
(タイムゾーン)		 now, parse, at, local, utc タイムゾーンを考慮した日時処理。
ActiveSupport::MessageEncryptor
(暗号化)		 encrypt_and_sign, decrypt_and_verify データを安全に暗号化・復号化し、改ざんを検出。
ActiveSupport::HashWithIndifferentAccess
(柔軟なハッシュ)		 [], fetch, merge, except, slice シンボルと文字列のキーを区別しないハッシュ。
